注意：

  • 初次接触 linux 系统的读者阅读本文之前请阅读安装linux的理论基础；
  • 本文讨论的内容只适用于桌面端的 linux 普通用户；
  • 本人只是业余爱好者，如果有什么说的不合适的地方，求轻喷。

众所周知，linux 一般指是系统内核，而我们一般安装的是基于 linux 内核的发行版本，它集成了一堆的软件构建了一套复杂的生态。

linux的发行版

linux 究竟有哪些发行版可以参考GNU/Linux Distributions Timeline1。

主流的 linux 桌面发行版本介绍，以及桌面环境的介绍可以参考孤门寒雪的简书：主流Linux桌面版简介2。

我们选择使用非常流行的 manjaro3 系统，它是 arch4 的衍生版。 我不同意孤门寒雪的简书：主流Linux桌面版简介2中， arch 系发行版对新手不友好的的观点。 实际上 manjaro 对新手相当友好，不友好的是 arch。

为什么不建议用 ubuntu

推荐用 ubuntu 的用户一般有两个理由

  • ubuntu 对用户友好。

什么是用户友好？按照大部分人的观点来看，开箱即用，不用做过多的配置，是用户友好。 但是对中文用户来说，能达到这一点的 linux 发行版根本没有。 所有的发行版都有可能出现网卡驱动，显卡驱动等等各种问题，没有一个发行版能做到 windows 那样的兼容性。 实际上，在各种硬件的兼容性上，ubuntu 并不是做的最好的那一个发行版。

  • 认为 ubuntu 的用户最多，随便百度一下可以在网上找到问题的解决办法。

对于第二个理由，咋一看好像对，但实际上这并不是解决问题的最好方式。 ubuntu 中文用户多，这个可以承认，但要知道中文网络内容的质量是很低的。 举个例子，对新手来说，安装软件后如何设置 PATH 环境变量是最先遇到的问题。 你到百度上搜一搜基本上都是让你到 ~/.bashrc 中加入一些 export 语句。 但有很多用户只知道这样做，却不知道为何这样做。 我有很多朋友用了好多年的 linux 都不太清楚 ~/.bashrc ~/.profile /etc/bash.bashrc /etc/profile 等等 shell 的配置文件到底有什么区别。

靠这些资料解决问题往往会陷入更大的问题，因为并不知其所以然啊。

为什么不推荐用 centos

推荐使用 centos 的用户一般只有一个理由， 稳定性。

这实际上是个伪命题。centos 可以很稳定，只要你别安装除了官方仓库中的软件。 这对于一般的桌面用户是不可能的，一但添加了第三方源，centos 就失去稳定的特性了。 而且，稳定性会更差，官方仓库中比较老的软件还有可能和第三方的软件冲突。

实际上，对于桌面用户来说，稳定性其实并不是很重要的事情，我倒建议桌面用户每隔一段时间就重装一次系统。 关于重装系统的问题我们接下来再谈。

为什么推荐使用 manjaro

  • manjaro 是 arch 的衍生版，相对于 arch，manjaro 更兼顾了用户友好。 manjaro 的安装和一般的发行版相同，开机就自带桌面等一系列软件。
  • manjaro 的滚动更新特性保证能用到最新的软件，并且 manjaro 的官方仓库软件要落后于 arch 一段时间，相当于 arch 用户为 manjaro 充当了小白鼠。 从而保证了 manjaro 的稳定性，不至于滚挂。
  • manjaro 是 arch 的衍生版，所以可以使用 AUR5 这一极其全面的第三方库。很多软件都可以优雅地一键安装，然后边喝咖啡边看旁边的 ubuntu 用户焦头烂额地配置依赖包，make install，配置环境变量等。
  • manjaro 是 arch 的衍生版，所以可以很好地利用 arch wiki6，这是所有 linux wiki 中最好用的。在这里你可以知道前面所说的中文博客中的那些操作的原理是什么，从而提高自己的知识水平。

有人可能会怀疑 manjaro 使用 AUR 的稳定性，但我认为对于桌面用户来说，稳定性不是最重要的，使用桌面 linux，做好本地数据的备份工作是一个很本职的工作。 而且，由于 AUR 的存在，很多软件甚至很专业的软件都可以在通过一个命令安装，比如 gmt，snaphu 等非专业人士根本没接触过的软件。 所以重装系统并不会占用过多的时间。

如果真的不想重装系统，在系统挂过之后，也可以通过 timeshift7 软件恢复系统，只要更改系统文件之后使用该软件做好系统备份。不过我还是觉得定期重装系统是个好习惯。

安装 linux

安装操作系统的逻辑是先在存储设备上安装一个小型的 live 系统，然后开机后进入这个系统，它的最主要的任务是安装真正的操作系统到该计算机的硬盘上，之后拔掉 U 盘，将 UEFI 设置为从该新安装的系统启动，这就完成新系统的安装了。注意这个小型的系统并非只有安装操作系统的功能，操作系统该有的基本功能它都有，所以在计算机出现某些问题后无法正确开机时，就可以通过这个系统修复。

下载 iso 官方镜像

首先到 manjaro 官网下载 iso 文件，国内的用户因为网速的原因请选择清华镜像。

制作启动 U 盘

然后使用 rufus 软件将 iso 文件 burn 到 U 盘中，注意 U 盘会被格式化，请做好数据备份。 如果可以的话不用使用太大的 U 盘，因为在 burn 时要检查整个 U 盘有没有坏块，容量大的检查比较慢。

如果你希望在一块 U 盘上 burn 多个系统的安装镜像，并且长期保存这些镜像以备日后修复或者重装系统[^10]，而且希望剩余的空间能够存储别的内容。可以将 U 盘多分出一个8g左右的分区，用来安装两个系统的安装镜像（亲测 8g 刚好够 Windows10 和 manjaro，有更多系统的需求可以适当增大），然后使用 yumi8 制作多系统引导。剩下的一个分区用来储存自己的文件。

一次使用 liveCD 解决计算机启动的例子。

之后的步骤可以参考 Seisman的博客：用 CentOS 7 打造合适的科研环境9中的安装步骤，我就不重复多说了。

注意如果是双系统安装，要把/boot/efi挂载到你 Windows 下的 EFI 分区。除此之外，不要对 Windows 系统所在的分区做任何的改动。

详细的安装教程，网上一搜一大堆，我就不说了。

安装之后的测试

安装完 linux 之后一定要测试各个硬件是否正常运行。通常可能出问题的有：

  • 蓝牙驱动；
  • 网卡驱动；
  • 显卡驱动，尤其是双显卡的计算机一定要做好测试；
  • 声卡驱动；
  • 能否正常睡眠和唤醒；
  • 亮度能否调节等。

其中最难搞的就是显卡驱动，双显卡用户一定要测试两块显卡能否正常工作。 Manjaro 默认使用开源的 video-linux 驱动，但是此驱动显然没有原厂驱动好用。 安装时可以选择安装闭源驱动，但我没找到在哪设置。 不过安装过后可以在 System Settings -> Hardware Configuration中更换。 记住，一定要选择 Auto Install Proprietary Driver，而不是自己随便挑一个。
