磁盘分区

磁盘需要分区使用。目前一共由两种分区方案。传统的 MBR 以及新式的 GPT。

MBR 方案

简单地说，磁盘相当于一块相当大的一维数组，需要分区使用。磁盘由很多个扇区组成，每个扇区是一个基本单元。磁盘的第一个扇区比较重要，主要记录了

  • 主引导记录 MBR：存储第一阶段的启动引导程序；
  • 分区表(partition table)：记录了整块硬盘的分区的状态，由于空间的限制，最多只能分四个分区.

为了解决这种限制，可以把这四个分区(当然也可以是一个两个或三个)中的一个继续划分。没有继续划分的分区称为主分区(primary partition)，继续划分的分区称为扩展分区(extended partition)。扩展分区继续划分的分区称为逻辑分区(logical partition)。

GPT 方案

GPT 是一种新式的磁盘分区方案，其作用和 MBR 类似但比 MBR 占用更多的空间，兼容 MBR ，只有主分区，且没有数量限制，建议使用。

关于磁盘分区的相关内容可以参考ArchWiki:Partitioning1。

关于MBR的相关内容可以参考Wikipedia:主引导记录2。

关于GPT的相关内容可以参考Wikipedia:全局唯一标识分区表3。

没有兴趣搞懂这些只想让自己电脑能正常工作的知道有这两种方案就行了。

启动引导程序 boot loader

boot loader 是用来识别操作系统的文件系统并读取系统内核加载到内存的程序。

常见的 boot loader 有常用于 linux 引导的 GRUB ，Windows 专用的 Windows 启动管理器。

boot loader 主要有三个功能：

  • 加载内核文件启动操作系统；
  • 将启动管理权转交给其他的 boot loader；
  • 提供不同的选项供选择。 即 boot loader 启动后，你可以选择加载可以选择的某一个操作系统的内核。这为同一台电脑安装多个系统提供了技术支撑。

关于 GRUB 的相关内容可以参考ArchWiki:GRUB4。

文件系统

文件系统是在存储设备上组织文件的方法。存储设备可以看成一个很长的数组，如何在这个数组上组织文件，比如形成树状的目录结构，依赖于文件系统。所以本质上说，文件系统就是一种数据结构。

目前 Windows 主要使用 NTFS 文件系统，linux 主要使用 Ext4 文件系统。操作系统可以支持各种各样的文件系统，比如linux 几乎支持所有的文件系统。所以能在 Windows 系统下使用的 NTFS 文件系统的U盘，可以直接在 linux 系统的机器上使用。

每个分区有自己的文件系统，定义了该分区存储空间的组织方式。

有时候，我们把可被挂载的一个对象称为一个文件系统。

文件系统都会保留一块启动扇区(boot sector),用来提供操作系统安装 boot loader。而操作系统会其根目录所在的文件系统的启动扇区写入一个boot loader 用来引导该操作系统的启动。这就解释了为什么制作启动 U 盘时必须要格式化整个 U 盘，因为需要通过格式化划定新的启动扇区，并写入 U 盘里的用来安装新系统的小型操作系统的 boot loader。

关于文件系统的详细描述可以参考鸟哥的linux私房菜:第七章，linux磁碟与档案系统管理5。

计算机开机过程

当按下开机键时，首先启动 BIOS ，它储存在主板的一块闪存上，和硬盘相互独立。新型的 BIOS 称为 UEFI。UEFI主流都支持 MBR 和 GPT 分区方案。

UEFI + GPT 是目前最先进的方案。

关于 BIOS 的详细描述可以参考Wikipedia:BIOS6中的相关内容。

关于 UEFI 的详细描述可以参考ArchWiki:UEFI7中的相关内容。

下面以一个 Windows-linux 双系统的例子说明整个过程。

你可以在 Windows 下的磁盘管理中找到一块 EFI 分区，文件系统为 FAT32，里面存储了操作系统的引导程序。

开机后，UEFI会主动寻找 EFI 分区，然后按照某一顺序读取 EFI 分区中的 boot loader。linux 下的 EFI 分区挂载在 /boot/efi 中，以我的计算机为例：

这里的grubx64.efi就是 grub 程序。

当只安装 Windows 系统时，EFI 分区中只有 Windows 的启动分区。当再安装一个 linux 系统时，会向 EFI 分区中写入 grub 程序，安装时能够修改 UEFI 的默认设置，首先启动 grub。而且安装时 grub 会识别 Windows 的 boot loader，并加入到自己的选项当中。

所以当开机后，UEFI 会运行 grub 程序，grub 程序会提供进入 linux 还是 Windows 的选项，选择希望的选项，就会加载相应的系统内核到内存中，计算机就启动完毕了。

grub 实际上是跨平台的 boot loader。
