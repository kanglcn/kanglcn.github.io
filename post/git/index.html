git文件管理

git init #初始化
git add [file] #把工作区的修改提交到暂存区
git commit -m "..." #把暂存区的内容提交到master分支并做评论
git diff [file] #查看区别
git log #commit的日志
git reset --hard HEAD^ #返回当前版本的上一个版本，也可以直接用commit id，
#git reset 之后会把工作区和暂存区中的所有改变清空，返回上一个版本，返回后暂存区是空的。
git reflog #记录命令的日志
git diff HEAD -- readme.txt #命令可以查看工作区和版本库里面最新版本的区别 和上面的相同
git checkout -- [file] #回到最近一次git commit 或者 git add 时的状态，适用于撤销没有add的修改
git reset HEAD [file] #丢弃暂存区相对于master版本库的修改，适用于add之后的修改，回到了上一个问题
git rm [file] #删除文件，只add，没有commit的文件不能删除
#git rm 之后要 commit

注意，git 不会管理文件夹，但会管理当前目录子目录中的文件。

git 远程仓库与 github

1
2
3
4
5
6
7
8
9

git remote add origin git@github.com:kanglcn/blog.git #关联远程仓库
# 当然也可以关联两个以上的仓库，只是名字要不同，不要都叫 master 就行了
git remote rm origin # 删除远程仓库
git remote -v #查看远程仓库的信息
git push -u origin master #关联后第一次推送master分支的所有内容
git push origin master #推送master分支
# 以上三条适用于将本地内容推送到远程空仓库中，如果不是空仓库，使用下一条命令
git clone git@github.com:kanglcn/blog.git ./blog #克隆远程仓库并自动关联，推送不用加 -u
git clone --recursive git@github.com:kanglcn/blog.git ./blog #克隆子模块

git 分支管理

git branch kanglcn #增加分支
git checkout kanglcn #切换分支
git switch kanglcn #切换分支 用switch更人性化一点
git checkout -b kanglcn #增加并切换分支
git switch -c kanglcn #增加并切换分支
git branch #查看分支
git branch -r #查看远端分支
git branch -a #查看所有分支
git branch -d kanglcn #删除kanglcn分支
git merge kanglcn #使master和kanglcn分支合并，可能会起冲突

注意当 merge 有冲突的时候，git 会在该文件中有冲突的那一行的位置保留两个版本这一行的内容，手动更改该文件，然后 add commit 即可。注意有冲突的时候不会使用 fast forward。

1
2
3
4

git merge --no-ff -m "..." kanglcn #禁用fast forward，将kanglcn分支merge到master上。
# 使用fast forward模式看不到历史上的merge，不用则可以看到。而且创建一个新的commit
git log --graph #可以看到分支合并图
git rebase #把分叉历史整理成直线

git branch --set-upstream-to=origin/kanglcn kanglcn #建立当前分支与远端分支的联系

git submodule 子模块

git 允许使用子模块子模块的日常使用请参考Maonx的简书：Git submodule 子模块的管理和使用3。

如果在本地建立含有 submodule 的 repo，直接推送该 repo 会使 github 上的 repo 没有关联到 submodule，因为 submodule 没有推送到 github 上。

如果先推送 submodule 到 github，再推送该 repo，这样的话 github 上的 repo 并不知道 submodule 的网址，这可以在 .gitmodule 文件中看到，该文件只会记录本地的 submodule 地址，不会记录远程的 submodule 网址，也达不到想要的效果。

最简单的办法是首先建立子模块并推送到 github 上。然后：

1

git submodule add git@github.com:kanglcn/blog.content.git ./content

然后再 push，这样就使 github 上的相应的 repo 成为另外一个的 submodule。
