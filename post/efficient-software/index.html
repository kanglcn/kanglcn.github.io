和linux 下的常用软件不同，本文讲述 linux 下的那些可以不用，但使用后让我们工作效率更高的软件。

注意：这些东西只是工具，不要在这些花里胡哨的东西上浪费太多时间。

fish

linux 下有很多种 shell，一般预装的都是 bash。 fish1 是一款很强大的 shell，和 bash 相比，有很多很好的特性。

安装 fish

1

sudo pacman -S fish

使用 fish

fish 的优点有很多

  • 彩色提示；
  • 自动建议；
  • 语法简单；
  • 自动补全等。

fish 的简单教程可以参考阮一峰的网络日志：Fish shell 入门教程2。

注意 fish 的语法和 bash 不兼容，不能成为 bash 的替代品。

vi 模式

1

fish_vi_mode

可以在终端下使用 vi 的编辑方法编辑命令。

我的 fish 使用方法

我只在交互模式下 使用 fish，这样可以利用 fish 的提示等优点，在写脚本时还是用 bash，因为要考虑兼容性。

有人觉得 zsh + oh-my-zsh 是更好的选择，我倒不这么认为。

  • fish 开箱即用，zsh 需要很多配置，比较麻烦，熟悉我的人都知道我是个相当懒的人；
  • 在终端下 zsh 和 fish 表现类似，zsh 并不比 fish 好用多少；
  • 使用 oh-my-zsh 会拖慢反应速度；
  • 因为脚本肯定会使用 bash，所以 zsh 对 bash 较好的兼容性只是鸡肋；
  • 我只在交互式终端下使用 fish 所以学习成本并不高。

ranger

ranger 是一款在终端下使用的文件管理器。

ranger 的操作逻辑和快捷键和 vim 很相似，使用h，j，k，l来移动光标，同时也支持gg，G这种快捷键。 ranger have four configuration files. For rc.conf and commands.py, ranger read both the global and user’s config. For scope.sh and rifle.conf ranger read either the users and global config.

File Operations

key binds operations
yy copy
pp paste
yd copy directory
yn copy name
yp copy path
dd cut
ya/da copy append
<SPACE> select file for multi file operations
gn add a new tab
gt/gT switch to tab
<Alt>+N switch to tab
t tag a file
cw rename
:bulkname rename multi files

alias, echo, mkdir, chmod can be used by command :command. :shell cmd will run the cmd. Note that the output will be displayed in the terminal where ranger is run. :shell nvim ... is a good command to open a new file and edit.

fzf

fuzzy finder

Using CTRL+hjkl to navigate. Enter key to select item. On multi-select mode -m, TAB to mark multiple files.

1

fzf > selected

It will write the selected item to selected.

1
2
3

vim $(fzf)
# or in fish
vim (fzf)

It will open the selected items through vim. But a better operation is using fzf completion. The configuration is in mybashrc file.
