linux 系统的初步配置

  2020-04-08   linux   manjaro linux 0  评论 字数统计: 2352字 阅读时长: 5分

本文谈谈安装完 linux 系统之后，如何打造合适的工作环境。

首先更新软件

sudo pacman -Syyu

pacman 是 arch 系的包管理软件，关于 pacman 的详细信息可以参考 ArchWiki:pacman1。

AUR

Arch 除了 pacman 管理的四个官方仓库外，还有第三方管理的仓库 Arch User Repository 即 AUR 。关于 AUR 的相关信息可以参考ArchWiki:Arch User Repository2。

注意 AUR 中的软件没有官方源中的安全，请谨慎使用。

AUR 可以通过 yay 这一工具管理。其使用方法和 pacman 相同。

sudo pacman -S yay

安装 vim 编辑器

需要安装编辑器来完成后续操作，所以先安装 vim。

sudo pacman -S vim

vim 使用 Shift + 鼠标选中，shift + 鼠标中键可以实现复制和粘贴的功能。

配置镜像源

国外的镜像在中国很慢，所以我们要用中国的镜像。推荐使用清华的镜像。

pacman 的配置文件安装在

/etc/pacman.conf

建议注释掉 Color，这样可以将不同类型的提示用不同的颜色表示。在这个文件中我们可以发现

[core]
SigLevel = PackageRequired
Include = /etc/pacman.d/mirrorlist

[extra]
SigLevel = PackageRequired
Include = /etc/pacman.d/mirrorlist

[community]
SigLevel = PackageRequired
Include = /etc/pacman.d/mirrorlist

# If you want to run 32 bit applications on your x86_64 system,
# enable the multilib repositories as required here.

[multilib]
SigLevel = PackageRequired
Include = /etc/pacman.d/mirrorlist

这是 manjaro 的四个官方软件仓库，我们可以添加第三方的仓库，比如包含很多中国软件的 archlinuxcn，通过 pacman 管理。在该文件中加入

[archlinuxcn]
SigLevel= TrustedOnly
Server = https://mirrors.tuna.tsinghua.edu.cn/archlinuxcn/$arch

这里直接使用了清华的镜像源，当然也可以使用其他的。安装完 archlinuxcn 之后一定要安装 archlinuxcn-keyring，导入 GPG-key。我也不知道为啥要装，让装就装吧。

sudo pacman -Syyu
sudo pacman -S archlinuxcn-keyring

注意 archlinuxcn 和 AUR 一样，只保证对 arch 的兼容，不支持 mnajaro，需要谨慎使用。 不过我已经说过，桌面版稳定性不太重要，而且我用了半年多没出过一次错。

在上面可以看到

Include = /etc/pacman.d/mirrorlist

这是 manjaro 三个官方仓库的镜像地址。vim 打开这个文件，可以发现有一系列的镜像地址，把我们希望的搬到文件最顶端

## Country : China
Server = https://mirrors.tuna.tsinghua.edu.cn/manjaro/stable/$repo/$arch

这样就可以将 core,extra,community 的源设置为清华镜像。这也可以通过命令

sudo pacman-mirrors -i -c China -m rank

搞定，但添加 archlinuxcn 还要手动搞。

如果希望更改为美国镜像，那就通过命令

sudo pacman-mirrors -i -c United_States -m rank

添加 archlinuxcn 的配置也要改为美国的镜像

[archlinuxcn]
SigLevel= TrustedOnly
Server= https://cdn.repo.archlinuxcn.org/$arch

修改 AUR 镜像

yay --aururl "https://aur.tuna.tsinghua.edu.cn" --save

yay --aururl "https://aur.archlinux.org/" --save

查看 yay 配置

yay -P -g

这部分废话这么多主要是因为 manjaro 的 wiki 比较垃圾，又不能照搬 arch 官网的原因。

虽然不能照搬，但 ArchWiki:Mirrors3 中的内容还是可以用来参考，毕竟只是二者只是官方仓库不同，逻辑和使用的工具还是一样的。 ArchWiki:Official repositories4 同样也可以用来参考。manjaro 的 official repositories 相比于 Arch 的延长了测试期，更安全。

关于 archlinuxcn 的相关信息可以参考 Arch Linux中文社区5。 关于 archlinuxcn 的 mirrorlist 可以参考 GitHub:archlinuxcn/mirrorlist-repo6。

本地化

更改时区

一般在安装系统的时候这部分就已经设置好了。如果想更改的话可以用 GUI 的工具，也可以用下面的命令。

timedatectl set-timezone Asia/Shanghai

当然也可以设置成别的。如果中国用户使用 manjaro-Windows 双系统，通常会碰到二者时间相差8小时的问题。这个问题我觉得对我影响不大，就凑合用了。

关于系统时间以及如何解决时差，可以参考 ArchWiki:System time7。

更改语言设置

Locales 被 glibc 和其它需要本地化的应用程序和库用来解析文本(或正确的显示当前区域的某些文字样式,如货币,时间,日期,特殊字符和其他的区域格式)。

具体如何设置请参考 ArchWiki:Locale8 以及 ArchWiki:Localization/Simplified Chinese[^11]。

安装和更改中文字体

sudo pacman -S wqy-microhei wqy-microhei-lite wqy-bitmapfont wqy-zenhei ttf-arphic-ukai ttf-arphic-uming adobe-source-han-sans-cn-fonts adobe-source-han-serif-cn-fonts noto-fonts-cjk

以上的字体可以依据需求选择，然后在GUI界面的字体设置中更改喜欢的字体。

关于中文字体的设置也可以参考 ArchWiki:Localization/Simplified Chinese[^11]。

输入法

建议使用fcitx。

Fcitx (Flexible Input Method Framework) ──即小企鹅输入法，它是一个以 GPL 方式发布的输入法平台,可以通过安装引擎支持多种输入法，支持简入繁出，是在 Linux 操作系统中常用的中文输入法。它的优点是，短小精悍、跟程序的兼容性比较好。

详细内容请参考 ArchWiki:Fcitx9。

sudo pacman -S fcitx-im #包含fcitx以及qt，gtk模块
sudo pacman -S fcitx-configtool #配置的GUI程序
sudo pacman -S fcitx-googlepinyin #google拼音，不建议用搜狗拼音。

运行 fcitx-configtool，添加已经下载好的googlepinyin。

为注册输入法模块，在/etc/profile(对所有用户有效)中插入

export GTK_MODULE=fcitx
export QT_IM_MODULE=fcitx
export XMODIFIERS="@im=fcitx"

这几个环境变量有什么用我也不清楚，不出问题的情况下，照做即可。想进一步了解可以参考 ArchWiki:Fcitx9。这几行对于WPS office中文输入至关重要。

如果想了解/etc/profile是什么东西，可以参考 ArchWiki:Bash10。

常用软件

sudo pacman -S google-chrome #chrome 浏览器
sudo pacman -S netease-cloud-music #网易云音乐
sudo pacman -S wps-office #WPS office
sudo pacman -S ttf-wps-fonts #WPS的一些字体
sudo pacman -S wps-office-mui-zh-cn #WPS的中文语言包
sudo pacman -S visual-studio-code-bin #VS code
sudo pacman -S git
sudo pacman -S lazygit
sudo pacman -S filezilla
sudo pacman -S fish # 一个用户友好的shell，注意不兼容bash
sudo pacman -S texlive-most
sudo pacman -S texlive-langchinese  #texlive的中文支持
sudo pacman -S texstudio
sudo pacman -S texmacs #一个所见即所得的编辑器

设置texstudio的 default compiler 为 xelatex，这是目前最好用的tex发行版。

一些经验之谈

linux，尤其是 arch 系发行版，有一种魔力，可以让你浪费很多时间配置桌面以使他更酷炫。请不要入这个坑。有时间还是要干些别的。

在workspace behavior中设置双击打开文件更符合习惯。
